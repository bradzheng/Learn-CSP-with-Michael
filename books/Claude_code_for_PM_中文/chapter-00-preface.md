## 本书适合谁

本书适合那些负责产品但无法阅读其代码库的产品经理。

如果你曾经等待工程团队回答"这个功能是怎么工作的？"，或者花了数小时手动整合本可以用程序化方式处理的客户反馈，那么本书将向你展示一条更好的路径。

你不需要专业地编写代码，也不需要深厚的命令行经验。你需要接受一个理念：**对于某些 PM 任务，让一个具有文件系统访问权限的智能体（agent）来执行，比与对话式 AI 交流更高效。**

## 你将学到什么

读完本书后，你将能够：

- 为每项 PM 任务选择合适的 AI 工具（Claude.ai vs. Claude Code）
- 安全地安装和配置 Claude Code，并建立成本控制
- 独立调查代码库以解答问题
- 将研究和反馈综合为结构化的产物（artifact）
- 为重复性 PM 任务构建可复用的工作流（技能/skill）
- 通过 MCP（模型上下文协议）连接你的 PM 工具栈（Jira、Slack、Figma）
- 在使用 AI 工具的同时与工程团队高效协作

## 本书不是什么

本书不是一本 Claude Code 综合手册，不是技术参考书，也不是面向工程师的。

本书严格聚焦于 PM 的使用场景。如果某个内容不能解决你实际面临的问题，它就不会出现在这里。

## 如何使用本书

- **如果你是 Claude Code 新手**：阅读第一部分（基础篇），了解工具选择和设置，然后跳到解决你最紧迫需求的那个部分。
- **如果你已经在使用 Claude Code**：根据你是在做代码调查还是工作流自动化，跳到第二部分（代码库智能）或第四部分（技能）。
- **如果你持怀疑态度**：阅读第 1 章。如果决策框架没有引起共鸣，那么这个工具目前可能还不适合你。

每章都包含你可以直接复制的精确提示词、成本估算和常见失败模式。你可以在页边做笔记，跳过不适用的章节，日后再回来看。

## 关于时效性的说明

AI 工具发展迅速。本书聚焦于持久的概念——工具选择的决策框架、可复现工作流的结构、与工程团队协作的模式——同时提供随着 Claude Code 变化可能需要调整的具体示例。

当功能变化时，原则不变：为任务选择合适的工具，控制成本，优雅地处理失败，尊重工程团队的专业知识。
