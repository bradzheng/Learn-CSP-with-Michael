# 第 5 章 市场与竞争分析

## 5.1 为什么 Claude Code 在研究方面胜过 Claude.ai

你的利益相关者要在周五前拿到一份竞争分析。你有三个选择。第一：花两天手动将信息汇编到 Google 文档中，下个季度就过时了。第二：用 Claude.ai 快速对话，产出不错但关闭标签页就消失了。第三：用 Claude Code 生成结构化、可版本化的产物，存放在你的代码仓库中，可以系统性地更新。

对于一次性问题（"竞品 X 的定价模式是什么？"），Claude.ai 就够了。对于需要持久化、可在 PRD 中引用、每季度更新、跨团队共享的研究，Claude Code 是正确的工具。区别不在于能力，而在于持久性和可重复性。

**为什么用 Claude Code 而非 Claude.ai 做研究？**

- **持久性。** 研究输出成为仓库中的文件。它们在会话关闭后仍存在，可以通过文件路径在文档中引用，并与团队的现有工作流集成。你今天生成的竞争矩阵，下个月 VP 要求再看时作为 `research/competitive-analysis-2026-01.md` 就在那里。
- **文件输出。** Claude Code 撰写结构化文档：markdown 表格、比较矩阵、格式一致的列表。Claude.ai 给你需要复制粘贴和重新格式化的文本。Claude Code 产出可直接提交的产物。
- **可重复性。** 你构建一次研究方法论（提示词、输出格式、分析框架），然后持续一致地执行。季度竞争审查使用相同的流程，使结果跨时间可比较。你不需要每个周期都重新发明研究结构。
- **可版本化。** Git 中的研究产物追踪变更。你看到 Q1 和 Q2 竞争分析之间的变化。你可以 diff 定价比较来发现趋势。这对向利益相关者展示"竞争格局如何变化"很重要。

**网页端 Claude 何时足够**：探索性研究，你还在弄清该问什么问题。快速事实核查。头脑风暴研究方向。任何对话比产物生成更有价值的场景。Claude.ai 在迭代对话方面表现出色。用它来思考，用 Claude Code 来记录。

**研究工具选择**：Claude.ai 用于探索性对话 vs. Claude Code 用于结构化、版本化的产物。

生成竞争分析的会话花费 $0.30-0.75，耗时 15-25 分钟。对比两天的体力劳动或过时研究导致糟糕产品决策的成本。如果你每个季度不止做一次市场研究，ROI 是立竿见影的。

## 5.2 构建可更新的竞争情报

你需要知道你在与谁竞争、他们提供什么以及你的差异化何在。手动研究涉及浏览竞品网站、做笔记、试图将所有东西保存在一个不一致且不完整的电子表格中。Claude Code 将其系统化。

**设置竞争分析工作区。** 为研究创建目录结构：

```bash
mkdir -p research/competitors
cd research/competitors
```

这使研究与代码分离，易于查找，可以按日期或主题组织。结构可能是：

```
research/
├── competitors/
│   ├── competitive-matrix-2026-01.md
│   ├── competitor-a-profile.md
│   ├── competitor-b-profile.md
│   └── competitor-c-profile.md
├── pricing/
└── market-sizing/
```

在仓库根目录以文件写入权限启动 Claude Code：

> **提示词模板：结构化竞品档案。** 有用的竞争分析的关键是一致性。每个竞品档案应以相同格式回答相同问题。定义你的结构，然后反复执行。

> **提示词：竞品档案**
>
> 使用网络搜索研究 [竞品名称]，在 `research/competitors/[competitor-name]-profile.md` 创建结构化档案。
>
> 包含以下部分：
> - **概览**：一段话描述他们做什么、服务谁、公司阶段
> - **产品服务**：核心产品和关键功能（列表）
> - **定价模式**：如何收费、有哪些层级、大致价格
> - **目标市场**：卖给谁（公司规模、行业、用户角色）
> - **关键差异化**：他们在定位中强调什么（3-5 条）
> - **优势**：相对于我们，他们强在哪里
> - **劣势**：他们在哪里弱或有缺口（3-5 条）
> - **近期动态**：产品发布、融资、合作（过去 6 个月）
> - **最后更新**：[今天日期]
>
> 使用网络搜索收集当前信息。保持每部分简洁（这是参考文档，不是深度分析）。

Claude Code 搜索网络获取信息，综合发现结果，撰写结构化 markdown 文件。输出生成需要 5-10 分钟，花费 $0.15-0.35，取决于网络搜索量。

你得到什么：一份一致、结构化的档案可供你参考。每个竞品都有相同的部分，使对比直观。当 CEO 问"竞品 X 与我们有什么不同？"时，你打开他们的档案，阅读差异化部分。

**输出格式：比较矩阵和功能表。** 单独的档案很有用，但利益相关者想要对比。一旦有了 3-5 个竞品档案，生成一个矩阵：

> **提示词：竞争矩阵**
>
> 基于 `research/competitors/` 中的竞品档案，在 `research/competitors/competitive-matrix-2026-01.md` 创建一个比较矩阵。
>
> 创建一个 markdown 表格比较：
> - 核心功能（行：认证、权限、集成、移动端支持、API 访问、报表等）
> - 竞品（列：我们、竞品 A、竞品 B、竞品 C、竞品 D）
> - 值：✓（完全支持）、~（部分/有限）、x（不可用）、?（未知）
>
> 包含一个定价比较表：
> - 行：免费层、入门级、专业级、企业级
> - 列：相同竞品
> - 值：价格/月 或 "联系销售" 或 "N/A"
>
> 在每个表格后添加简短分析（2-3 句话），突出关键模式。

Claude Code 读取你已有的竞品档案，提取相关信息，格式化为可快速浏览的表格。矩阵展示缺口：竞品有你没有的功能、你定价不具竞争力的层级、你应该强调的优势。

**将结果存储为可版本化的产物。** 将这些文件提交到 git：

```bash
git add research/competitors/
git commit -m "Add Q1 2026 competitive analysis"
```

现在你的研究有版本控制了。三个月后，更新档案并生成标有 Q2 的新矩阵。diff 文件查看变化：

```bash
git diff competitive-matrix-2026-01.md competitive-matrix-2026-04.md
```

你看到竞品 A 新增了移动功能，竞品 B 提价了，一个新竞品进入了市场。这个历史视图展示格局演变，而不仅仅是当前状态。

**竞争格局映射的成本和时间。** 初始设置（5 个竞品档案 + 矩阵）：45-60 分钟，80,000-150,000 token，$0.40-0.75。季度更新（刷新档案 + 重新生成矩阵）：20-30 分钟，40,000-70,000 token，$0.20-0.35。对比每个周期 8-12 小时的手动研究。

**此方法何时失效**：公开信息稀缺的利基行业。定价不透明的竞品（一切都要"联系销售"）。信息每周就过时的快速变化市场。在这些情况下，Claude Code 仍然结构化你所知的内容，但你需要用销售电话、分析师报告或客户访谈来补充。

## 5.3 无需电子表格追踪竞品定价

定价情报需要系统性收集和结构化比较。你的竞品收费 $49/月，但包含什么功能？什么限制？与你的 $59/月层级相比如何？手动研究捕获了定价但遗漏了上下文。Claude Code 两者兼得。

**系统性地收集定价情报。** 定价页面会变。促销会上线。折扣会出现。你需要一个可以重复的快照方法论，以便随时间追踪变化。

> **提示词：定价深度分析**
>
> 使用网络搜索研究 [竞品名称] 当前的定价模式，在 `research/pricing/[competitor-name]-pricing-2026-01.md` 创建详细分解。
>
> 包括：
> - **定价层级**：名称、月价格、年价格（如不同）
> - **每层功能限制**：用户数、存储、API 调用、项目数等
> - **关键功能门槛**：哪些功能被锁定在更高层级？
> - **附加项**：可选额外功能及其定价
> - **折扣**：年付折扣百分比、批量折扣、非营利/教育定价
> - **免费层**：包含什么、有什么限制
> - **企业层**：定价公开还是"联系销售"？包含什么？
> - **计费模式**：按用户、按功能、按用量、固定费率？
>
> 使用网络搜索收集当前信息。如果某事没有公开记录，标注为"未披露"。
> 在顶部添加"最后更新"日期。

Claude Code 搜索定价页面，提取细节，一致地结构化。你得到一份远比"他们收费 $49/月"详细的定价分解。你知道那 $49 包含什么、不包含什么，以及与其他层级相比如何。

**提示词模板：定价模式比较。** 一旦有了单个竞品定价文档，比较它们：

> **提示词：定价比较表**
>
> 基于 `research/pricing/` 中的定价文件，在 `research/pricing/pricing-comparison-2026-01.md` 创建比较表。
>
> 创建表格：
> - 行：定价层级（免费、入门/基础、专业/标准、企业）
> - 列：我们、竞品 A、竞品 B、竞品 C
> - 值：每用户每月价格（或如果不是按用户计费，则为月度总价）
>
> 在定价表下方，创建功能限制比较：
> - 行：关键限制因素（用户数、存储、API 调用、项目数）
> - 相同列
> - 值：实际限制（如："10 用户"、"100GB"、"1000/天"）
>
> 添加分析：
> - 每个层级哪个竞品最便宜/最贵？
> - 我们在哪里定价有竞争力 vs. 溢价 vs. 折扣？
> - 每个价格点的价值主张是什么？

这揭示了定价策略差异。也许竞品 A 标价更便宜但按用户算更贵。也许竞品 B 在相同价格提供更慷慨的限制，表明你应该重新考虑你的上限。也许你是唯一有真正免费层的，这是你没有足够强调的竞争优势。

**通过信息传达审计分析定位。** 定价不只是数字，而是竞品如何定位价值。他们强调什么？他们声称解决什么问题？他们引用什么客户类型？

> **提示词：定位分析**
>
> 使用网络搜索研究 [竞品名称] 的网站，在 `research/competitors/[competitor-name]-positioning.md` 分析其定位。
>
> 从其首页、产品页面和营销内容中提取和分析：
> - **主标题**：他们的主要价值主张是什么？
> - **关键信息主题**：什么概念反复出现？（如："快速"、"安全"、"简单"、"企业级"）
> - **引用的客户类型**：他们说他们服务谁？
> - **问题陈述**：他们解决什么客户痛点？
> - **差异化声明**：什么让他们"不同"或"更好"？
> - **社会证明**：他们展示什么案例研究、Logo、客户评价？
> - **行动号召**：他们希望访客做什么？（试用、演示、联系销售）
>
> 提供 2-3 句话总结其定位策略。

你了解到竞品 A 定位为"面向受监管行业的企业级安全"，而竞品 B 定位为"最快上手的方式（无需设置）"。这些是不同的价值主张，吸引不同的客户。你的定位需要要么与两者区分开来，要么瞄准一个服务不足的细分市场。

**用标注日期的产物追踪变化。** 定位会变化。竞品在进入新市场时、融资轮次后或应对竞争时会重新定位。追踪这些揭示了策略。

将定位分析按日期保存：`competitor-a-positioning-2026-01.md`、`competitor-a-positioning-2026-04.md`。当你比较版本时，你看到信息传达的演变。也许竞品 A 不再提"小团队"，开始强调"企业规模"。他们正在向高端市场移动。这为你自己的定位决策提供信息。

**定价分析的时间和成本**：一个竞品的定价深度分析：10-15 分钟，20,000-35,000 token，$0.10-0.18。5 个竞品的定价比较表：10 分钟，15,000-25,000 token，$0.08-0.13。每个竞品的定位分析：15-20 分钟，25,000-40,000 token，$0.13-0.20。

**何时失效**：复杂定制化定价的竞品，公开信息不反映实际成交。不透明的企业定价，没有列出的数字。在这些情况下，用销售情报补充 Claude Code 研究：潜在客户告诉你的竞品报价、赢单/丢单分析、与前竞品客户的对话。

## 5.4 创建经得起推敲的市场估算

利益相关者想要数字。这个市场有多大？多少潜在客户？我们的可触达机会是什么？市场估算需要假设、数据来源和结构化的推理。Claude Code 不会魔法般地产生准确的数字，而是结构化你的方法论，使你的假设可见且经得起质疑。

**使用 Claude Code 结构化 TAM/SAM/SOM 分析。** 总可触达市场（TAM）、可服务可触达市场（SAM）、可服务可获得市场（SOM）：这是经典的风投框架。你需要估算这些、记录你的推理并使其可被审查。

> **提示词：市场估算框架**
>
> 帮我使用 TAM/SAM/SOM 框架估算 [产品/类别] 的市场规模。创建 `research/market-sizing/market-size-2026.md`。
>
> 对每个指标提供：
> - **定义**：这个指标对我们的产品意味着什么
> - **计算方法**：我们将如何估算它
> - **数据来源**：数字从哪来（行业报告、人口普查数据、代理指标）
> - **假设**：我们假设什么是真实的
> - **估算范围**：低/中/高估算
> - **推理**：为什么这个计算是合理的
>
> 结构：
> 1. **TAM** — 总可触达市场（如果每个可能使用此产品的人都使用了）
> 2. **SAM** — 可服务可触达市场（我们能服务的现实子集）
> 3. **SOM** — 可服务可获得市场（我们在 1-3 年内可以捕获的）
>
> 展示你的工作过程。使假设明确且保守。

Claude Code 帮助你理清逻辑：

```
## 总可触达市场 (TAM)
**定义**: 北美所有拥有 50-5000 名员工、需要 [产品类别] 的公司。
**计算方法**:
- 北美拥有 50-5000 名员工的公司总数：~850,000（美国人口普查局，2024 年）
- 需要 [产品类别] 的比例：~30%（行业报告显示类似工具的渗透率）
- 公司 TAM：850,000 × 30% = 255,000 家公司
**收入 TAM**:
- 假设平均交易规模为 $5,000/年
- 255,000 家公司 × $5,000 = $12.75 亿
**假设**:
- 公司规模范围捕获了目标市场（排除了过小和过大的企业）
- 30% 的需求基于类似产品采纳率
- $5,000/年是现实的平均值（SMB 和中等市场的混合）
**估算范围**:
- 低：$8.5 亿（20% 需求，$4K 均值）
- 中：$13 亿（30% 需求，$5K 均值）
- 高：$21 亿（40% 需求，$6K 均值）
```

你不是把假设藏在电子表格里，而是文档化且可被审查。当你的 CFO 质疑 30% 渗透率时，你可以明确讨论并更新模型。

**提示词模板：带假设的市场估算。** 关键是记录方法论，而不是产生单一的"魔法数字"。

> **提示词：市场估算方法论审查**
>
> 审查 `research/market-sizing/market-size-2026.md` 中我的市场估算假设。对每个假设评估：
> - 数据来源可信吗？
> - 逻辑合理吗？
> - 有没有明显的缺口或有问题的跳跃？
> - 什么敏感性分析会有帮助？（如果这个假设偏差 50% 会怎样？）
>
> 提供建设性反馈以加强分析。

Claude Code 挑战你的推理。它可能指出："30% 渗透率假设没有来源，仅描述为'行业报告显示'。你能引用具体报告或提供代理计算吗？"或者："计算假设所有目标公司花费相同金额，但 SMB vs. 中等市场定价可能差异显著。考虑分段计算。"

这种反馈帮助你产出经得起推敲的市场估算，而不只是听起来不错的数字。

**细分框架及其应用。** 市场不是铁板一块。你有按公司规模、行业、地域或用例划分的细分市场。细分显示该聚焦哪里。

> **提示词：市场细分**
>
> 按 [公司规模 / 行业 / 地域 / 用例] 细分 [产品类别] 市场。在 `research/market-sizing/segmentation-2026.md` 创建细分分析。
>
> 对每个细分估算：
> - **规模**：多少潜在客户？
> - **特征**：什么定义了这个细分？
> - **需求**：他们最关心什么？
> - **付费意愿**：价格敏感度（高/中/低）
> - **竞争**：还有谁瞄准这个细分？
> - **我们的适配度**：我们今天服务这个细分的程度如何？（强/中/弱/无）
>
> 按吸引力排序：规模 × 付费意愿 × 我们的适配度。

你得到一份有清晰推理的优先排序细分列表。也许"中等市场医疗"得分高，因为它大、愿意为你拥有的合规功能付高价、且被聚焦于 SMB 或企业的竞品服务不足。这为市场进入策略提供信息。

**为利益相关者审查记录方法论。** 市场估算文档成为参考。你的董事会演示引用它们。你的销售团队用它们理解目标市场。记录方法论使数字可信。

在你的市场估算文档中加入"如何更新此分析"部分：

```
## 如何更新此分析
刷新这些估算：
1. 查看美国人口普查局数据中更新的公司数量
2. 查看 [具体行业报告] 中更新的渗透率
3. 根据实际销售数据验证平均交易规模
4. 用更新的输入重新运行计算
5. 更新"最后审查"日期
**审查节奏**：每年一次，或进入新市场时。
```

这使其他人（或将来的你）可以在不重新发明方法论的情况下更新分析。

**时间和成本**：初始市场估算分析：30-45 分钟，50,000-80,000 token，$0.25-0.40。细分分析：20-30 分钟，30,000-50,000 token，$0.15-0.25。用新数据更新：15-20 分钟，20,000-35,000 token，$0.10-0.18。

**局限性**：Claude Code 没有专有市场数据。它使用公开可用的信息，这意味着估算可能较粗糙。对于高风险决策如融资或并购，用付费分析师报告、调查或顾问来补充。对于产品策略和内部规划，Claude Code 的结构化方法已经足够。

## 5.5 在行业趋势中分离信号与噪音

行业趋势塑造产品策略。AI 能力在进化。隐私法规在收紧。客户期望在转变。你需要追踪趋势、将炒作与信号分开、综合对你的路线图的影响。这是受益于 Claude Code 综合和文档能力的研究工作。

**从多个来源聚合趋势数据。** 趋势不是单一文章，而是跨来源的模式。你阅读分析师报告、技术博客、客户对话、会议演讲。将其综合为连贯的趋势简报非常耗时。Claude Code 将其系统化。

创建趋势工作区：

```bash
mkdir -p research/trends
```

将来源（粘贴文章文本、复制报告部分、总结对话）收集到单个文件中：

```
research/trends/
├── ai-tooling-gartner-2026.txt
├── privacy-regulations-summary.txt
├── customer-feedback-themes.txt
└── saas-trends-2026.txt
```

然后让 Claude Code 综合：

> **提示词：趋势综合**
>
> 审查 `research/trends/` 中的来源文件，将关键主题综合为 `research/trends/trend-brief-2026-01.md`。
>
> 对每个识别出的主要趋势：
> - **趋势名称**：简洁标签
> - **描述**：正在发生什么变化（2-3 句话）
> - **证据**：什么来源支持此观点？（引用具体文件）
> - **时间**：这正在、即将还是将来发生？（1 年 / 2-3 年 / 3+ 年）
> - **对我们的相关性**：这如何影响我们的产品？（高 / 中 / 低）
> - **影响**：我们应该考虑做什么？（2-3 条）
>
> 按相关性排序。限制在前 5-7 个趋势，避免简报过于庞杂。

Claude Code 读取你的来源文件，识别反复出现的主题，产出结构化的趋势简报。你得到的是综合，而不仅仅是聚合。

**提示词模板：行业趋势综合。** 关键是将趋势与行动连接起来。

示例输出：

```
## 趋势：AI 驱动的生产力功能正在成为基本门槛
**描述**: 客户越来越期待产品中内置 AI 辅助：自动完成、建议、智能默认值。18 个月前还是差异化优势的功能，正在成为 SaaS 工具中预期的基线功能。
**证据**:
- Gartner 报告（ai-tooling-gartner-2026.txt）指出 67% 的 B2B 买家期望 AI 功能
- 客户反馈（customer-feedback-themes.txt）包含 12 处提到"为什么这不建议..."或"竞品有 AI 做这个"
- SaaS 趋势报告（saas-trends-2026.txt）将 AI 功能列为前 3 投资领域
**时间**: 正在发生。客户今天已在基于 AI 能力比较产品。
**对我们的相关性**: 高。我们 AI 功能很少，而竞品正在推出。
**影响**:
- 评估哪些工作流最受益于 AI 辅助
- 优先考虑改善核心工作流的 AI 功能，而非新奇功能
- 考虑 AI 能力的合作 vs. 自建
- 监控可能与 AI 相关的"缺失功能"导致的客户流失
```

这在来源之间连接各个点，使趋势可操作。你不是仅仅报告"AI 很热"，而是在展示证据、时间线和对产品具体的影响。

**分离趋势报告中的信号与噪音。** 不是每个趋势都重要。有些是炒作，有些是真实但与你的市场无关，有些重要但在遥远的未来。Claude Code 帮助你过滤。

> **提示词：趋势验证**
>
> 审查 `research/trends/trend-brief-2026-01.md` 中的趋势。对每个趋势评估：
> - **信号 vs. 噪音**：这有多个可信来源支持，还是只是一篇文章的猜测？
> - **相关性过滤**：这影响我们的目标客户细分，还是其他市场？
> - **紧迫性**：我们需要本季度、今年行动，还是只需监控？
>
> 标记任何看似没有实质内容的炒作趋势。建议哪些趋势值得立即纳入路线图考虑 vs. 长期监控。

这防止追逐每个光鲜的趋势。你得到现实检验。"'万物区块链'趋势出现在一个来源中，没有客户验证。建议降级。'隐私优先架构'趋势出现在法规分析、客户反馈和竞争定位中。这是信号。建议路线图审查。"

**创建可操作的趋势简报。** 输出应该推动决策，而不是躺在文件夹里。结构化趋势简报以供利益相关者消费：

> **提示词：高管趋势摘要**
>
> 基于 `research/trends/trend-brief-2026-01.md`，在 `research/trends/trend-summary-exec-2026-01.md` 创建一页高管摘要。
>
> 格式：
> - **我们必须响应的前 3 个趋势**：简短描述 + 推荐的具体行动
> - **需要监控的趋势**：2-3 个新兴但尚不紧迫的趋势
> - **我们忽略的趋势及原因**：1-2 个不适用于我们的趋势
>
> 保持可快速浏览。高管应能在 3 分钟内阅读并理解影响。

这成为你策略审查中的一页幻灯片、路线图规划中的参考或执行团队的简报。它以行动为导向，而不仅仅是信息性的。

**时间和成本**：将 5-8 个来源文档综合为趋势简报：20-30 分钟，40,000-70,000 token，$0.20-0.35。从趋势简报创建高管摘要：10 分钟，15,000-25,000 token，$0.08-0.13。趋势分析周期总计：30-40 分钟，$0.30-0.50。

**何时刷新**：快速变化行业每季度，较慢市场每年。重大行业事件发生时临时更新（新法规、主要竞品动作、技术突破）。

**局限性**：Claude Code 综合你给它的东西。如果你的来源质量低或偏向某个视角，综合也会如此。垃圾进，垃圾出。将 Claude Code 的综合与多样化的可信来源结合：分析师报告、客户研究、行业专家、竞争情报。
