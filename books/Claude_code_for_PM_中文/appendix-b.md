# 附录 B：SKILL.md 模板

创建新技能时复制此模板。完整示例见第 9 章，设计模式见第 8 章。

## 最小模板

```markdown
---
name: my-skill-name
description: 一句话描述此技能做什么，以及何时自动触发。
---

# [技能名称]

## 目的
[此技能完成什么——一段话，最多]

## 输入
- [预期此技能读取的文件或数据]

## 流程
1. [步骤 1：读取和验证输入]
2. [步骤 2：核心操作]
3. [步骤 3：格式化和保存输出]

## 输出格式
[生成的 markdown 结构]

## 质量标准
- [具体且可衡量的标准]
```

## 全面模板

```yaml
---
name: skill-name
description: 清晰描述技能功能。包含触发词，便于 Claude 在用户请求匹配时自动调用。例如："从 CSV 或 JSON 文件合成客户反馈为主题报告。当用户提到反馈分析、NPS 合成或支持工单主题时自动调用。"
---
```

正文包含：

1. **目的**：用户得到什么（而非技能内部做什么），一段话最多
2. **预期输入**：具体格式。"CSV 文件"太模糊；"文本列中包含反馈的 CSV"告诉 Claude 该找什么
3. **流程步骤**：按编号顺序执行。每一步可验证（"检查文件是否存在"、"识别相关列"）
4. **输出格式规范**：确切结构。"`# [报告标题]` → `## 主题：[名称]` → `- 频率：X%`"
5. **质量标准**：具体、可衡量。"每个主题必须有 2+ 条支持引用"、"表格必须使用一致的列标题"
6. **已知边缘情况或限制**："不处理图像数据"、"期望 CSV 使用 UTF-8 编码"

## 字段参考

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 是 | 唯一标识符（kebab-case）。匹配文件名。 |
| `description` | 是 | 单行摘要。包含触发词用于自动调用匹配。 |
| 目的 | 是 | 领先用户获得什么的价值 |
| 流程 | 是 | 编号步骤。具体、可验证的指令 |
| 输入 | 推荐 | 期望的具体文件格式和结构 |
| 输出格式 | 推荐 | 输出 markdown 的确切结构 |
| 质量标准 | 推荐 | 可衡量的验证标准 |
| 边缘情况 | 可选 | 诚实说明限制 |

## 各部分指南

- **目的**：一段话最多。领先用户获得什么，而非技能内部做什么。
- **预期输入**：具体格式。"CSV 文件"太模糊；"文本列中包含反馈的 CSV"告诉 Claude 该找什么。
- **流程**：可验证的步骤。"读取输入文件"是模糊的；"检查 `data/feedback/` 中是否存在 CSV 文件"是可执行的。
- **输出规范**：明确指定结构。Claude 越是必须猜测，输出就越不一致。
- **质量标准**：可衡量的。"质量好"是模糊的；"每个主题必须有 2+ 条支持引用"是可验证的。

## 常见错误

- **描述缺少触发词**：Claude 将你的请求与技能描述进行匹配。如果你的描述不包含用户实际使用的词语，自动调用永远不会触发。
- **流程步骤不可验证**：如果 Claude 无法判断某个步骤是否完成，它将跳过或猜测。
- **输出格式太模糊**：如果不指定结构，每次运行会产生不同的格式。
