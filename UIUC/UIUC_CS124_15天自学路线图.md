## 1. 平台与课程简介

### 关于 learncs.online

这是 UIUC CS 124 的**官方免费在线版本**，由 Prof. Geoffrey Challen 创建。疫情期间开发后被证明非常有效，已成为 CS 124 的永久授课形式。特点：

- ✅ **完全免费**，无需注册 UIUC 即可使用所有课程材料
- ✅ **纯浏览器运行**，无需安装任何 IDE 或环境
- ✅ **交互式 Playground**：每节课都有可编辑、可运行的代码示例
- ✅ **多讲师讲解**：同一概念有 2+ 种不同讲解方式，一种没懂可以换一种
- ✅ **调试练习**：每节课有专门的 debug 挑战，训练找错能力
- ✅ **代码质量反馈**：不仅判对错，还评价代码风格和效率

### 课程使用语言

| 语言 | 特点 |
|------|------|
| **Kotlin**（主推荐） | CS 124 的主要教学语言，现代、简洁、与 Java 100% 互通 |
| **Java**（备选） | 同样完整覆盖，适合习惯 Java 的学习者 |

> 💡 **建议**：选择 **Kotlin** 路线。原因是：(1) CS 124 以 Kotlin 为主；(2) Kotlin 更简洁，减少 boilerplate 干扰；(3) UIUC 后续 CS 128 会用 C++，语言转换是必经之路，Kotlin 的好习惯（类型推断、空安全）对后续学习有帮助。

### 2.2 心态准备

| 原则 | 说明 |
|------|------|
| **接受不理解** | 某个概念没搞懂是正常的——切换到另一位讲师的解释，或先标记、第二天回看 |
| **敲代码 > 看代码** | 每个 Playground 都要自己动手改、动手跑，不要只看不动 |

| **不求完美** | 15 天不可能掌握到和修完一学期课同样的深度。目标是**建立完整的知识框架**，为 CS 128 做好充分准备 |

## 3. 15天总体节奏设计

### 阶段性规划

```
阶段一：编程基础 (Day 1-4)
  变量 → 运算 → 条件 → 数组 → 循环
  约 18-20 节课 | 节奏：4-5 节/天
  
阶段二：函数与算法思维 (Day 5-8)
  函数 → 调试 → 算法 → 字符串 → 集合
  约 16-18 节课 | 节奏：4-5 节/天

阶段三：面向对象编程 (Day 9-12)
  对象 → 类 → 构造器 → 继承 → 多态 → 接口 → 异常
  约 14-16 节课 | 节奏：3-4 节/天

阶段四：数据结构与算法 (Day 13-15)
  链表 → 树 → 递归 → 排序 → 综合复习
  约 9-11 节课 | 节奏：3 节/天 + 复习
```

### 5.2 遇到困难时的应对策略

| 困难类型 | 应对 |
|----------|------|
| **概念不理解** | (1) 换讲师 (2) 看 Java 版本的解释 (3) Google "kotlin [概念] tutorial" (4) 标记后跳过，第二天回看 |

| **脑子很乱** | 停止 coding，拿纸笔画出数据流、变量关系和逻辑流程 |
| **进度落后** | 不要跳过练习只追课程。宁可少看 2 节课也要做完练习 — 编程是"做"会的，不是"看"会的 |

## 6. 常见问题 (FAQ)

### Q1: 15天真的能学完一学期的内容吗？

**A**: 15天可以**建立完整的知识框架和编程能力**，但深度无法完全等同于在一个学期内慢慢消化。关键区别：
- ✅ 你能做到：理解所有核心概念、写出中等复杂度的程序、为 CS 128 做好充分准备
- ⚠️ 需要后续补充：大量练习积累的"码感"、多人合作项目经验、面对全新问题的独立解决能力
- 💡 建议：开学后 CS 128 开始前，每周做 2-3 道 LeetCode Easy 题维持手感

### Q2: 选 Kotlin 还是 Java？

**A**: 强烈建议 **Kotlin**。原因：
1. learncs.online 的 Kotlin 内容是 Prof. Challen 亲自设计的，最完整
2. Kotlin 代码量通常比 Java 少 30-40%，减少 boilerplate 干扰
3. UIUC CS 124 以 Kotlin 为主
4. UIUC CS 128 切换到 C++，所以无论学 Kotlin 还是 Java，都要学新语言——重要的是理解编程思维

### Q3: 某个概念实在搞不懂怎么办？

**A**: (1) 标记 → (2) 跳过 → (3) 继续往下走。很多概念在后续课程中会自然建立联系。例如：
- 函数没完全理解？继续写更多函数就会明白
- 递归搞不懂？先接受，Day 14 在树的语境下会"突然通了"
- 面向对象很抽象？Day 11 的多态练习会帮你内化

### Q4: 每天应该做多少作业题？

**A**: learncs.online 每节课自带 1 个编程/调试练习 — 这是底线。此外：
- 重点关注本文档中"今日重点练习"部分
- 如果当天时间紧张，优先完成编程练习而非追赶课程进度
- 每个阶段最后做一次综合项目（本文档已设计）

### Q5: 如何评估自己是否"学会了"？

**A**: 对每节课，问自己三个问题：
1. 能用自己的话解释这个概念吗？（Feynman 技巧）
2. 不看代码，能写出这个概念的示例程序吗？
3. 能想到这个概念在实际程序中的应用场景吗？

如果三个问题都能回答"是"，那么这节课你已经掌握了。

### Q6: 学完 CS 124 之后呢？

**A**: 标准 UIUC CS 学习路径：
```
CS 124 (Kotlin) → CS 128 (C++) → CS 173 (离散数学) + CS 225 (数据结构，C++)
```
- CS 128 可以用 [cs128.org](https://cs128.org) 的材料预习
- CS 173 和 CS 225 是 CS+X 转专业的关键评估课程
- 暑假学完 CS 124 后，如果还有时间，可以开始预习 CS 128
