## 概述（Overview）

你好！欢迎来到伊利诺伊 CS 128 能力考试信息网站。在继续之前，你**必须**先熟悉 [Illinois CS Proficiency Exams 网站](https://cs.illinois.edu/) 上的内容。除了关于伊利诺伊能力考试流程的关键信息外，CS 128 能力考试的日期、时间和地点也会在那里发布。

## 考试（The Exam）

CS 128 能力考试是一场**三小时**的综合考试，涵盖一个典型学期中 CS 128 的所有内容。

### 题型
- **选择题**（Multiple-choice）
- **匹配题**（Matching）
- **填空题**（Fill-in-the-blank）
- **自由回答题**（Free-response）—— 这些题型**只有一次**作答机会
- **编程题**—— 可以尝试**任意次数**以获得满分

我们会在考试环境内酌情提供库的文档。

### 考试地点
你将在 **计算机化考试中心（CBTF）** 参加考试。请务必查看 [CBTF 的政策与流程](https://cbtf.illinois.edu/)。

### 学术诚信
注册考试即表示你同意以下声明：
> "我不会在此学术工作中给予或接受任何未经授权的帮助。我特别声明，我不会接受或提供任何违反本能力考试精神或字面规定的帮助。"

如果被抓到作弊或被发现参与作弊，你将（毫无疑问地）**考试不及格**，收到 **FAIR 违规记录**，并且在注册 CS 128 后将**立即不及格**。注册考试时，你需要同意此条款并确认我们制定的其他学术诚信政策。

**想都不要想作弊。** 我们对此非常严肃，并要求你认真对待学习。

## 考试主题（Topics）

### 命令行与 Unix 工具
- 能够使用基本的 bash shell 命令在命令行中导航
- 能够使用基于 Unix 的工具管理、组织、编译简单复杂度的 C++ 项目：
  - **版本控制（git）**
  - **构建系统（make）**
  - **Clang C++ 编译器**

### 编辑与调试工具
- **Microsoft Visual Studio Code**
- **GNU 调试器（GDB）**
- **Valgrind**
- **Clang AddressSanitizer (ASan)**
- **Clang MemorySanitizer (MSan)**
- 使用 **catch2** 进行单元测试

### 库
- **catch2**

### C++ 编程能力要求

使用 C++ 编程语言，你应该准备好：

1. **编写解决各种问题的计算机程序**

2. **分解问题**，识别其重要特征，并设计/实现解决该问题的计算机程序；阐述各种设计和实现决策的理由

3. **阅读、编写和理解**命令式、声明式、面向对象和递归式计算机代码 —— 判断意图、追踪运行时行为、识别可能发生的错误

4. 使用**行为驱动设计**和**测试驱动开发**，根据规范推导/编写程序

5. 使用基本的**可视化编辑**和**命令行调试工具**来开发和调试程序

6. 详细说明**基本类型**的表示差异及其含义

7. 使用**基于字符的 IO** 和**基于二进制的 IO 流**从标准输入和文件读取；写入标准输出和文件；使用流状态从格式化错误中恢复

8. 应用**面向对象编程概念**：对象、类、继承、多态和封装

9. 使用**函数模板**和**类模板**实现和理解泛型编程解决方案

10. 区分程序地址空间的区域，并理解它们对**对象生命周期**的支撑能力

11. 展示对类中**动态分配对象的内存管理**的充分理解，包括：
    - **零法则**（Rule of Zero）
    - **三法则**（Rule of Three）
    - **五法则**（Rule of Five）
    - **内存泄漏**（Memory leaks）
    - **悬空指针**（Dangling pointers）

12. 使用动态内存实现本课程涵盖的**数据结构**——包括向量、列表、树和图

13. 正确理解**函数式编程**的基本概念及其在 C++ 语言中的应用

14. 设计基本的**数据库模式**并转换为 SQLite；编写使用适当适配器与这些数据库交互的代码

15. 熟悉**二叉树**和**无限分支树**

16. 使用动态内存实现尊重**二叉搜索树性质**的二叉树；定义遍历、查询和其他在 BST 上操作的算法

17. 熟悉**图结构**；理解图术语

18. 使用**邻接表**和**邻接矩阵**实现有向/无向图；编写允许以这两种表示进行图探索的算法

### 设计模式
- 定义什么是**设计模式**以及它们的用处
- 充分描述和/或实现以下模式：
  - **工厂模式**（Factory）
  - **迭代器模式**（Iterator，适用于线性和非线性数据结构）
  - **中介者模式**（Mediator）
  - **观察者模式**（Observer）
  - **策略模式**（Strategy）

### 智能指针
- 详细说明什么是**智能指针**以及它们如何解决与原始指针相关的"问题"，特别是**资源拥有权**
- 比较和对比 **`std::unique_ptr`**、**`std::shared_ptr`** 和 **`std::weak_ptr`**；`std::make_unique` 和 `std::make_shared`
- 实现一个**朴素的** `std::unique_ptr`、`std::shared_ptr` 和/或 `std::weak_ptr`
- 使用智能指针实现本课程涵盖的数据结构——包括向量、列表、树和图
