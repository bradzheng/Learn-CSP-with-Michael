# UIUC CS 173 + CS 225 课程详解

> 适用对象：UIUC CS+GGIS ICT 转专业规划中的大一新生
>
> 更新日期：2025 年 6 月（基于 2024-2026 学年课程信息）
>
> 两门课均为 ICT 申请的 **必须前置课**，每门成绩须达 **B+ 及以上**。

## CS 173 — 离散数学（Discrete Structures）

### 基本信息

| 项目 | 详情 |
|------|------|
| 学分 | 3 |
| 前置课 | CS 128（≥ B-）+ 一门微积分 |
| 讲授教授 | Margaret Fleck / Benjamin Cosman 等 |
| 教材 | *Building Blocks for Theoretical Computer Science*（Fleck 自编，免费电子版） |
| 性质 | **理论/证明课，不是编程课** |
| 课程页面 | [CS 173 Fall 2025 大纲](https://courses.grainger.illinois.edu/cs173/fa2025/ALL-lectures/syllabus.html) |

### 课程内容

| 模块 | 主题 |
|------|------|
| 逻辑 | 命题逻辑、谓词逻辑、量词 |
| 证明方法 | 直接证明、反证法、归纳法（弱归纳/强归纳）、结构归纳 |
| 集合论 | 集合运算、幂集、关系、等价关系、偏序 |
| 函数 | 单射/满射/双射、函数复合、可逆性 |
| 数论 | 模运算、欧几里得算法、RSA 加密简介 |
| 图论 | 图的基本定义、路径、连通性、二分图、树 |
| 递归 | 递归定义、递推关系、递归树 |
| 算法分析 | Big-O/Ω/Θ 记号、时间复杂度 |
| 形式语言 | 正则表达式、DFA、NFA、文法 |
| 可数性 | 可数/不可数集合、对角论证法 |
| NP 完备 | P vs NP 概念入门 |

### 考试形式（这课最特殊的地方）

**核心制度：Examlet（小考）**

| 项目 | 详情 |
|------|------|
| 频率 | 约 **每周 1 次**，共 12 次 + 1 次期末 |
| 时长 | 50 分钟/次，期末 110 分钟 |
| 地点 | **CBTF**（Computer-Based Testing Facility）线下机考 |
| 平台 | **PrairieLearn**（需输入 LaTeX 数学公式） |
| 允许资料 | **Closed-book**，无笔记、无电子设备 |
| 预约 | 自行在线预约 time slot，热门时段抢很快 |
| Drop 政策 | 通常只能 **drop 1 次** examlet 分数 |
| 重考机会 | 期末可参加 **1 次**历史 examlet 重考，分数上限 80%，取较高分 |

### 评分权重

| 组成部分 | 权重 |
|------|------|
| **Examlets（含期末）** | **~90%** |
| Moodle 阅读测验 | 5-10% |
| Discussion 作业 | 5% |

> ⚠️ **成绩几乎完全取决于考试表现**，这就是 CS 173 压力最大的地方。

### 字母等级线（近似，可能视整体调整）

| 总分 | 预期等级 |
|------|------|
| ≥ 90% | A-/A |
| ≥ 80% | B-/B |
| ≥ 70% | C-/C |
| ≥ 50% | D |

> 对应 ICT 要求：需要 **≥ 80%（B+ 或 B 上沿附近）**

### 难度评估

| 维度 | 评价 |
|------|------|
| 内容难度 | ⭐⭐⭐（数学概念本身不深） |
| 考试压力 | ⭐⭐⭐⭐⭐（非常高——每周一考，只允许 drop 1 次） |
| 运气成分 | ⭐⭐⭐⭐（考题可能来自旧题也可能全新，波动大） |
| 容错率 | ⭐⭐（12 次考试只 drop 1 次，一次崩了很难救） |
| 拿 B+ 难度 | ⭐⭐⭐⭐（需要持续稳定发挥，无法考前突击） |

### 学生真实反馈

- "考试占总分 90%，12 次只能 drop 1 次，容错率极低"
- "有时候考旧题就很简单，有时候出全新题完全懵了"
- "LaTeX 输入数学公式是必须学的，从第 3 次 examlet 起强制在 PrairieLearn 里打公式作答"
- "不是 coding 课，是数学证明课——要做好写证明的心理准备"
- "有 skills list 和 study problems，认真练可以考好"
- "即便努力，最终成绩很大程度取决于运气"
- "有人能拿到超过 100% 的分数（通过 bonus 和学习态度），但也有人刷题很多仍然考不好"

### 关键提醒

1. 开学就要熟悉用 **LaTeX 在 PrairieLearn 里打数学公式**
2. 每周跟紧——这是马拉松不是冲刺，不能堆到考前
3. 善用 Fleck 的教材 + skills list + study problems
4. CBTF 抢 slot：收到通知第一时间预约，否则只剩凌晨 1 点这种时段
5. **C- 以下被认为不足以进入 CS 225**，ICT 要求 B+
6. 刷历年 examlet 题有帮助，但不能只背题——考试经常出新题

## CS 225 — 数据结构（Data Structures）

### 基本信息

| 项目 | 详情 |
|------|------|
| 学分 | 4 |
| 前置课 | CS 128 **且** CS 173 |
| 语言 | **C++** |
| 性质 | **编程 + 理论结合**，CS 专业最重要的基础课 |
| 课程页面 | [CS 225 Spring 2024 考试页](https://courses.grainger.illinois.edu/cs225/sp2024/exams/) |

### 课程内容

#### C++ 强化（前 3-4 周）

- 面向对象编程（类、继承、多态）
- 手动内存管理（指针、动态分配、内存泄漏、valgrind）
- 泛型编程（模板）
- 引用 vs 指针、二维指针

#### 核心数据结构

| 类别 | 具体 |
|------|------|
| 线性结构 | 链表（单向/双向/循环）、栈、队列 |
| 树 | 二叉树、BST、AVL 树、B 树、B+ 树、堆/优先队列 |
| 哈希表 | 哈希函数、冲突解决（链式/开放寻址） |
| 图 | 邻接矩阵/邻接表、DFS、BFS、最小生成树（Prim/Kruskal）、最短路径（Dijkstra） |
| 高级 | 并查集（Union-Find）、摊还分析、计算几何入门 |

#### 算法

- 排序（快排、归并、外部排序）
- 递归与分治
- 动态规划入门
- 二分查找
- 算法复杂度分析（Big-O、递推式、归纳证明）

### 作业构成

#### MP（Machine Problem）— 7 个编程作业（300 分）

| MP | 主题 | 说明 |
|----|------|------|
| MP1 | 图像操作 | C++ 热身，1 周 |
| MP2 | 图像与贴纸 | 链式结构入门 |
| MP3 | 图像与列表 | List ADT 实现 |
| MP4 | Flood Fill | 递归与图遍历 |
| MP5 | PhotoMosaic | 树/空间结构 |
| MP6 | 调试工具 | 软件工具链 |
| MP7 | 迷宫求解 | 图算法（收官大作） |

- 每个 MP ~50 分，共 300 分
- 每次提前交有 **+7 extra credit**
- 逾期 1 天交：93% 计分
- 每个 MP 大概需要 **1-2 周**完成

#### Labs — ~14 个实验（120 分，上限 100）

每周 1 次，10 分/次，多的变 extra credit。内容覆盖调试、内存、继承、栈/队列、树、Huffman 编码、AVL 树、B 树、哈希表、堆、图等。

### 考试形式

| 项目 | 详情 |
|------|------|
| **Exam 0** | 不计分诊断考（熟悉 CBTF 环境） |
| **期中** | 约 **6 次**，每次 70 分，CBTF 机考 |
| **期末** | 1 次，**160 分**，累计覆盖 |
| 总考试分 | 6×70 + 160 = **580 分** |

- 同样在 **CBTF** 进行，需要自己预约
- 含选择题 + 编程题 + 书面分析题

### 难度评估

| 维度 | 评价 |
|------|------|
| 内容难度 | ⭐⭐⭐⭐（数据结构概念 + C++ 实现双线并行） |
| workload | ⭐⭐⭐⭐⭐（每周 10-20 小时，MP 非常耗时） |
| 考试压力 | ⭐⭐⭐（6 次期中分散了压力，不像 173 每周都考） |
| 拿 B+ 难度 | ⭐⭐⭐⭐（需要 MP + 考试均衡不崩） |
| 重要性 | ⭐⭐⭐⭐⭐（全 CS 项目最重要的基础课，实习面试的根本） |

### 关键提醒

1. **C++ 基础很重要**——CS 128 里好好学 C++，否则 225 开头几周就会崩溃
2. **MP 时间管理**：每个 MP 至少提前 1 周开始，deadline 前赶工 debug 到凌晨是常态
3. **MP 提前交有 EC**：积少成多，可能最终决定 A- 还是 B+
4. **内存泄漏是扣分重灾区**——valgrind 是你最好的朋友
5. 建议**不要 proficiency out**——除非已经把数据结构学透了，CS 225 是技术面试的基础
6. **POTD（每日一题）**坚持做——不仅是 EC 来源，更是考试练手的最佳资料

## CS 173 vs CS 225 对比总结

| | CS 173 | CS 225 |
|------|------|------|
| 学分 | 3 | 4 |
| 前置课 | CS 128 + 微积分 | CS 128 + **CS 173** |
| 类型 | 理论/证明 | 编程/数据抽象 |
| 语言 | 数学证明 / LaTeX | C++ |
| 考试频率 | 每周 1 次（12次） | 约每 2-3 周一考（6次） |
| 考试权重 | ~90% | ~58% |
| 编程作业 | 无 | 7 个 MP |
| 每周耗时 | 6-10h | 10-20h |
| 考试平台 | PrairieLearn | CBTF |
| ICT 要求 | B+ 及以上 | B+ 及以上 |
| 最大挑战 | 持续性考试压力 | MP 时间黑洞 + C++ 内存管理 |

| 容错率 | 低（只 drop 1 次） | 中（6 次 + MP + lab 分摊风险） |

## 给 CS+GGIS 转专业新生的策略建议

### 防坑攻略

1. **CS 173 比 CS 225 更早接触**（大一春即可修）。它看似内容简单但考试制度（examlets）让很多人崩在这里，**不要低估**。

2. **CS 225 的 workload 是 CS 128 的 2-3 倍**。那个学期尽量减轻其他课负担，不要同时排太多硬课。

3. **两门课的 CBTF 预约都要抢**。收到通知立刻登录预约，好的 time slot（下午、周末）几分钟就没了。

4. **LaTeX 数学公式输入**是 CS 173 的隐藏门槛——你既要会做证明，还要会快速打字输入。开学第一周就装好环境熟练起来。

5. **不要试图同一学期修 CS 173 + CS 225**。225 的前置就是 173，正常流程是大一春 173 → 大二秋 225。

6. **拿到 B+ 是底线不是目标**。ICT 竞争激烈，两门课都尽量冲 A-/A。

### ICT 关键节点参照

```
大一秋：CS 124 ────────────────（打基础）
大一春：CS 128 + CS 173 ──────（173 B+ 截点1 ☑️）
大二秋：CS 225 ──────────────（225 B+ 截点2 ☑️）
大二秋末/春初：提交 ICT 申请 ──（前提条件全部满足 ✅）
```

---

> **一句话**：CS 173 拼的是**每周稳定输出**（12 次 examlet 马拉松，容错率极低）；CS 225 拼的是**项目管理能力**（MP 耗时长 + 考试 + lab，时间分配决定成绩）。两门都需要拿到 B+ 以上才能符合 CS+GGIS 的 ICT 门槛。
