# 解题分析：16L 水平均分给 4 人

## 一、设计的第一部分：分析

### 1. 理解问题

**Goal：** 有 3 个桶（8L、8L、3L），初始状态是两个 8L 桶装满水（共 16L），3L 桶为空。要把 16L 水平均分给 4 个人，每人恰好 4L。4 个人有各自的接水容器，给出的水不能收回。需要实现一个**分水的操作步骤序列**。

**核心约束：**
- 只能操作 3 个桶（容量：8L、8L、3L）
- 4 个人有各自的容器，接走的水不能再倒回去（**不可逆操作**）
- 目标是让 4 个人各得到恰好 4L 水

## 二、设计的第二部分：Implementation

### 定结构 — 三大控制结构

#### 顺序结构
- 初始化状态（桶满、人空）
- 依次执行倒水操作
- 依次给人分水

#### 分支结构

倒水时需要判断：**能倒多少**（受目标桶剩余容量和源桶水量的限制）：

```
amount = Math.min(源桶水量, 目标桶空余容量)
```

给人分水时判断：**人还需多少**：

```
amount = Math.min(桶中水量, 目标4L - 人已接水量)
```

#### 循环结构

本题采用**显式步骤序列**而非 while 循环，因为每一步的目标人物和桶是确定的，不需要条件判断退出——所有步骤在编译时已规划好。

但若要用循环，Loop 3 要素为：
- **Init：** 所有人接水量为 0
- **Step：** 每次操作减少桶中水量，增加某人接水量
- **Exit：** `people[0]==4 && people[1]==4 && people[2]==4 && people[3]==4`

## 三、数学推理 — 核心解题策略

### 关键洞察

3L 桶是**量具**，通过反复用 3L 桶量水，配合接水杯中**已有的水量**，可以间接实现非 3 的整数倍的分量（如 1L、2L、4L）。

**原理：** 当接水杯已有 3L、还需 1L 时，从 3L 桶往杯中倒，杯子满了（倒了 1L）就停——此时 3L 桶中剩 2L。这就是间接得到 1L 和 2L 的方法。

### 详细方案推导

```
初始: [8, 8, 0]  人=[0, 0, 0, 0]

第1步 - 人0 得 4L:
  桶0(8) → 倒满桶2(3):    [5, 8, 3]
  桶2(3) → 给人0:          人0=3,     [5, 8, 0]
  桶0(5) → 倒满桶2(3):    [2, 8, 3]
  桶2(3) → 给人0(已有3,需1): 倒1L, 人0=4✓, [2, 8, 2]

第2步 - 人1 得 4L:
  桶2(2) → 给人1:          人1=2,     [2, 8, 0]
  桶1(8) → 倒满桶2(3):    [2, 5, 3]
  桶2(3) → 给人1(已有2,需2): 倒2L, 人1=4✓, [2, 5, 1]

第3步 - 人2 得 4L:
  桶2(1) → 给人2:          人2=1,     [2, 5, 0]
  桶1(5) → 倒满桶2(3):    [2, 2, 3]
  桶2(3) → 给人2(已有1,需3): 正好3L, 人2=4✓, [2, 2, 0]

第4步 - 人3 得 4L:
  桶0(2) → 给人3:          人3=2,     [0, 2, 0]
  桶1(2) → 给人3(已有2,需2): 正好2L, 人3=4✓, [0, 0, 0]

最终: 桶=[0,0,0], 人=[4,4,4,4] ✓  全部倒空，每人正好 4L
```

### 关键步骤对应关系

| 步骤 | 操作 | 桶状态 | 人状态 | 说明 |
|------|------|--------|--------|------|
| 初始 | — | [8, 8, 0] | [0, 0, 0, 0] | 两满一空 |
| 1 | 桶0→桶2 | [5, 8, 3] | [0, 0, 0, 0] | 3L桶满 |
| 2 | 桶2→人0 | [5, 8, 0] | [3, 0, 0, 0] | 人0得3L |
| 3 | 桶0→桶2 | [2, 8, 3] | [3, 0, 0, 0] | 桶0剩2L |
| 4 | 桶2→人0 | [2, 8, 2] | [4, 0, 0, 0] | **人0=4✓**，桶2剩2L |
| 5 | 桶2→人1 | [2, 8, 0] | [4, 2, 0, 0] | 人1得2L |
| 6 | 桶1→桶2 | [2, 5, 3] | [4, 2, 0, 0] | 3L桶满 |
| 7 | 桶2→人1 | [2, 5, 1] | [4, 4, 0, 0] | **人1=4✓**，桶2剩1L |
| 8 | 桶2→人2 | [2, 5, 0] | [4, 4, 1, 0] | 人2得1L |
| 9 | 桶1→桶2 | [2, 2, 3] | [4, 4, 1, 0] | 3L桶满 |
| 10 | 桶2→人2 | [2, 2, 0] | [4, 4, 4, 0] | **人2=4✓** |
| 11 | 桶0→人3 | [0, 2, 0] | [4, 4, 4, 2] | 人3得2L |
| 12 | 桶1→人3 | [0, 0, 0] | [4, 4, 4, 4] | **人3=4✓**，桶全空 |

## 四、Java 实现

### 核心设计

- **3 种操作封装为方法：** `pourBucket`（桶↔桶）、`pourToPerson`（桶→人，智能判断）、`emptyBucketToPerson`（清空桶给人）

## 五、总结

### 核心策略

本题的关键是**利用 3L 桶 + 接水杯剩余容量**来间接量取非 3 的倍数的水量：

```
接水杯已有3L → 倒水时只能再进1L → 3L桶剩2L
接水杯已有2L → 倒水时只能再进2L → 3L桶剩1L
接水杯已有1L → 倒水时还能进3L → 正好倒空
```

### 程序结构

```
WaterDivide
├── pourBucket(src, dst)     // 桶↔桶倒水，受容量限制
├── pourToPerson(bucket, person)  // 桶→人倒水，受目标量限制
├── solve()                   // 12步显式分水流程
└── verify()                  // 验证最终状态
```

### 控制结构使用

| 结构 | 使用方式 |
|------|----------|
| **顺序** | 12步按序执行 |
| **分支** | `Math.min()` 判断倒水量、剩余容量、还需水量 |
| **循环** | 未使用 — 步骤序列在编译时已确定 |
