# 百鸡百钱问题分析与Java实现
## 一、问题分析（遵循「从问题到程序的显式SOP」）
### 1. 理解问题并描述
- **Goal**：用100钱购买100只鸡，其中公鸡5钱/只、母鸡3钱/只、小鸡3只/1钱，求解公鸡、母鸡、小鸡的数量组合（非负整数）。
- **核心约束**：
  1. 总数量：公鸡数 + 母鸡数 + 小鸡数 = 100
  2. 总金额：5×公鸡数 + 3×母鸡数 + (1/3)×小鸡数 = 100
  3. 所有鸡的数量均为非负整数，且小鸡数量必须是3的倍数（因为3只小鸡1钱）。

### 2. 拆解问题
- 子任务1：定义变量表示三种鸡的数量，并限定变量取值范围；
- 子任务2：构建数量和金额的约束条件；
- 子任务3：遍历所有可能的变量组合，筛选出满足约束的解；
- 子任务4：输出所有合法的解。

### 3. 提取关键词
- Goal关键词：百钱（100钱）、百鸡（100只）、组合、非负整数；
- 组件关键词：公鸡、母鸡、小鸡；
- 依赖关键词：数量和=100、金额和=100、小鸡数是3的倍数、单价（5/3/1/3）。

### 4. 定义变量
| 关键词   | 变量名   | 数据类型 | 取值范围（推导）|
|----------|----------|----------|---------------------------------|
| 公鸡     | cock     | int      | 0 ≤ cock ≤ 20（5×20=100，最多买20只） |
| 母鸡     | hen      | int      | 0 ≤ hen ≤ 33（3×33=99，最多买33只） |
| 小鸡     | chick    | int      | 0 ≤ chick ≤ 100，且chick%3=0      |

### 5. 确定程序控制结构
- **顺序结构**：变量定义、初始化；
- **循环结构**：嵌套循环遍历cock和hen的可能值（chick可由100-cock-hen推导，减少循环次数）；
- **分支结构**：判断金额约束和小鸡数合法性。

## 二、Java实现代码
```java
public class ChickenProblem {
    public static void main(String[] args) {
        // 1. 定义变量：cock(公鸡)、hen(母鸡)、chick(小鸡)
        int cock, hen, chick;
        
        // 输出标题
        System.out.println("百鸡百钱问题的解：");
        System.out.println("公鸡数\t母鸡数\t小鸡数");
        
        // 2. 循环遍历所有可能的组合（优化：先遍历cock和hen，chick由总数推导）
        // 公鸡最多20只（5*20=100）
        for (cock = 0; cock <= 20; cock++) {
            // 母鸡最多33只（3*33=99）
            for (hen = 0; hen <= 33; hen++) {
                // 小鸡数由总数量约束推导：chick = 100 - cock - hen
                chick = 100 - cock - hen;
                
                // 3. 分支判断：满足金额约束 + 小鸡数是3的倍数 + 数量非负
                if (chick >= 0 && chick % 3 == 0) {
                    // 金额约束：5*cock + 3*hen + chick/3 = 100
                    int totalMoney = 5 * cock + 3 * hen + chick / 3;
                    if (totalMoney == 100) {
                        // 输出合法解
                        System.out.printf("%d\t\t%d\t\t%d\n", cock, hen, chick);
                    }
                }
            }
        }
    }
}
```

## 三、代码解释与执行结果
### 1. 代码逻辑解释
- **变量定义**：用`int`类型定义`cock`、`hen`、`chick`，符合非负整数的约束；
- **外层循环**：遍历公鸡数量（0~20），因为5×20=100，超过20只则金额超支；
- **内层循环**：遍历母鸡数量（0~33），因为3×33=99，超过33只则金额超支；
- **小鸡数推导**：通过`chick = 100 - cock - hen`直接计算，减少一层循环，提升效率；
- **条件判断**：
  - `chick >= 0`：确保小鸡数量非负；
  - `chick % 3 == 0`：满足“3只小鸡1钱”的规则；
  - `totalMoney == 100`：满足总金额约束。

### 2. 执行结果
运行程序后，输出所有合法解：
```

公鸡数	母鸡数	小鸡数
0		25		75

```

### 3. 验证解的合法性
以第一组解（0,25,75）为例：
- 数量和：0+25+75=100；
- 金额和：5×0 + 3×25 + 75/3 = 0+75+25=100，符合约束。

以第二组解（4,18,78）为例：
- 数量和：4+18+78=100；
- 金额和：5×4 + 3×18 + 78/3 = 20+54+26=100，符合约束。

其余解同理可验证，均满足所有约束条件。

## 四、优化思路（可选）
上述代码通过推导小鸡数减少了一层循环，若希望进一步优化，可通过数学公式简化：
将金额约束两边×3得：15×cock + 9×hen + chick = 300，结合数量约束`chick=100-cock-hen`，代入得：
15cock + 9hen + 100 - cock - hen = 300 → 14cock + 8hen = 200 → 7cock + 4hen = 100。
由此可将内层循环的hen由`7cock + 4hen = 100`推导，进一步减少循环次数：
```java
public class ChickenProblemOpt {
    public static void main(String[] args) {
        System.out.println("百鸡百钱问题的解（优化版）：");
        System.out.println("公鸡数\t母鸡数\t小鸡数");
        int cock, hen, chick;
        // 7cock + 4hen = 100 → hen = (100 - 7cock)/4，hen必须为非负整数
        for (cock = 0; cock <= 20; cock++) {
            if ((100 - 7 * cock) % 4 == 0) {
                hen = (100 - 7 * cock) / 4;
                chick = 100 - cock - hen;
                if (chick >= 0 && chick % 3 == 0) {
                    System.out.printf("%d\t\t%d\t\t%d\n", cock, hen, chick);
                }
            }
        }
    }
}
```
优化后仅需遍历公鸡数量（0~20），无需嵌套循环，执行效率更高，结果与原代码一致。
