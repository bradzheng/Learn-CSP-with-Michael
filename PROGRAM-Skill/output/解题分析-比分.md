# 解题分析：2位选手比赛胜负判定

## 一、设计的第一部分：分析

### 1. 理解问题

**Goal：** A、B 两位选手进行比赛，比较得分决出胜者。

**核心规则：**
- 得分高的选手获胜
- 得分相同时，后出手的选手获胜

**期望输出：** 给定双方的得分和出手顺序，返回胜者（A 或 B）。

---

### 2. 拆解问题

#### 包含关系（子任务分解）

| 子任务 | 说明 |
|--------|------|
| 比较双方分数 | 判断 A 和 B 谁的分数更高 |
| 处理分数不同的情况 | 分数高者直接获胜 |
| 处理分数相同的情况 | 比较出手顺序，出手晚者获胜 |

#### 依赖关系

```
输入(选手A得分, 选手B得分, 选手A出手顺序, 选手B出手顺序)
        │
        ▼
   比较 scoreA vs scoreB
        │
   ┌────┴────┐
   ▼         ▼
scoreA>scoreB  scoreB>scoreA   scoreA==scoreB
   │             │                │
   ▼             ▼                ▼
  返回'A'     返回'B'        比较 orderA vs orderB
                                   │
                              ┌────┴────┐
                              ▼         ▼
                          orderB>orderA  orderB<=orderA
                              │             │
                              ▼             ▼
                            返回'B'       返回'A'
```

---

### 3. 关键词提取

| 维度 | KEY WORD | 说明 |
|------|----------|------|
| **Goal** | 决出胜者 | 最终要返回赢家 |
| **组件** | 选手、得分、出手顺序 | 核心业务概念 |
| **依赖** | 比较、分支判断 | 先比分数，同分再比出手顺序 |

### 4. 定变量

| 关键词 | 变量名 | 数据类型 | 说明 |
|--------|--------|----------|------|
| 选手A的得分 | `scoreA` | `int` | 整数分数 |
| 选手B的得分 | `scoreB` | `int` | 整数分数 |
| 选手A的出手顺序 | `orderA` | `int` | 出手先后编号，数值越大表示出手越晚 |
| 选手B的出手顺序 | `orderB` | `int` | 出手先后编号，数值越大表示出手越晚 |
| 胜者 | `winner` | `char` | 返回 `'A'` 或 `'B'` |

**变量规则检查：**
- 每个变量单一职责，无歧义 ✓
- 数据类型选择合理（分数用 int，顺序用 int，结果用 char）✓
- 无冗余变量 ✓

## 二、设计的第二部分：Implementation

### 定结构 — 程序的三大控制结构

#### 顺序结构
- 接收输入参数
- 执行比较逻辑
- 返回结果

#### 分支结构（核心控制结构）

本题的核心是**分情况讨论**，映射到 `if-else`：

| 分支 | 条件表达式 | 含义 | 返回值 |
|------|-----------|------|--------|
| 分支1 | `scoreA > scoreB` | A得分高于B | `'A'` |
| 分支2 | `scoreB > scoreA` | B得分高于A | `'B'` |
| 分支3 | `orderB > orderA`（在分数相等时） | B出手比A晚 | `'B'` |
| 分支4 | `orderB <= orderA`（在分数相等时） | A出手比B晚或同时 | `'A'` |

**分支划分的基准：** 先比较分数大小（广义的关系运算），分数相同再比较出手顺序。

#### 循环结构
- 本题为单次比较场景，不涉及重复操作，**无需循环**。

---

## 三、Java 实现

### 核心方法

```java
/**
 * 判断比赛胜者
 * 规则：
 *   1. 得分高者胜
 *   2. 得分相同时，后出手者胜
 *
 * @param scoreA 选手A的得分
 * @param scoreB 选手B的得分
 * @param orderA 选手A的出手顺序（越大越晚）
 * @param orderB 选手B的出手顺序（越大越晚）
 * @return 'A' 表示A胜，'B' 表示B胜
 */
public static char determineWinner(int scoreA, int scoreB, int orderA, int orderB) {

    // 分支1：A得分更高 → A胜
    if (scoreA > scoreB) {
        return 'A';
    }

    // 分支2：B得分更高 → B胜
    if (scoreB > scoreA) {
        return 'B';
    }

    // 分支3：得分相同 → 出手更晚者胜
    // 出手顺序编号大的 = 出手更晚
    return (orderB > orderA) ? 'B' : 'A';
}
```

## 五、总结

本题的逻辑链条非常清晰：

```
先比分数 → 分出高低 → 返回高分者
   ↓ 同分
再比出手顺序 → 晚出手者胜
```

整个实现仅用了 **2个分支判断**（分数比较）+ **1个三元表达式**（同分时的出手顺序比较），没有使用循环，结构简洁明了。
